# Reproducibility Notes

## What is and is not included

This repo contains the code used for the thesis experiments. The raw CRSP/WRDS data is not included because it is proprietary. The GA result files (parquets in `results/ga/`, `results/benchmarks/`, etc.) are also not committed; they are generated by running the pipeline steps below.

The result figures and summary tables committed to `results/` were produced from a single full pipeline run on the data described in the README.

## Raw data required

Place these two files in `data/raw/` before running anything:

- `crsp_returns.csv` - CRSP monthly stock file (CIZ format, 2000–2025). Source: WRDS, under CRSP Annual Update / Stock Version 2 (CIZ) / Monthly Stock File. Required columns: `MthCalDt`, `MthRet`, `MthRetx`, `MthPrc`, `PrimaryExch`, `ShareType`, `SecurityType`, `SecuritySubType`, `USIncFlg`, `IssuerType`, `ShrOut`.
- `risk_free_rate.csv` - FRED DTB3 series (3-month T-bill, annual %). Download from FRED and save as-is.

## Full reproduction order

Run these commands in order from the repo root. Each step writes files that later steps read.

```bash
# 1. Parse and clean raw data
python3 -m src.data.loader
python3 -m src.data.risk_free_rate

# 2. Build monthly universe and returns
python3 -m src.data.universe
python3 -m src.data.returns

# 3. Run benchmarks
python3 -m src.benchmarks.mvo
python3 -m src.benchmarks.equal_weight

# 4. (Optional) re-tune hyperparameters; tuned values are already hardcoded
python3 -m src.optimization.optuna_tuner

# 5. Full GA run (35–45 minutes on a 16-core machine)
python3 -m src.optimization.runner

# 5b. K-sensitivity (each fixed K value run independently)
python3 -m src.optimization.k_sensitivity

# 6. Lambda ablation (about 30 minutes for the lambda=0 run)
python3 -m src.ablation.ablation_lambda

# 7. Generate all figures and tables
./run_evaluation.sh
```

A fast smoke test of the GA (10 periods, 3 runs, 50 generations) is available but still requires the processed data files:

```bash
python3 -m src.optimization.runner --debug
```

## Smoke / integrity tests (no data required)

The unit tests in `tests/` use only small synthetic data and can be run without WRDS access:

```bash
python3 -m pytest
```

These tests check that the GA operators, repair function, metrics, and backtest bookkeeping behave correctly on synthetic inputs. They do not reproduce the actual OOS results.

## Caveats

- The exact OOS numbers depend on the CRSP data vintage used. Minor differences in survivorship or return adjustments between WRDS data vintages can shift performance figures slightly.
- The Optuna tuning run is stochastic. Re-running tuning may produce slightly different hyperparameters and therefore slightly different GA results even on identical data.
- The GA itself is stochastic. The runner uses a fixed seed per period, so a re-run on identical data should produce identical results, but only if the same seed sequence is preserved in `runner.py`.
